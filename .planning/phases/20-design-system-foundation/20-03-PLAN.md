## Glass Tokens

Glass tokens enable the glassmorphism design aesthetic. They are designed to work on both light and dark backgrounds.

| Token | Light Mode | Dark Mode | Purpose |
|-------|------------|-----------|---------|
| `--glass-bg` | `oklch(1 0 0 / 60%)` | `oklch(0.15 0 0 / 60%)` | Standard glass panel background |
| `--glass-bg-heavy` | `oklch(1 0 0 / 70%)` | `oklch(0 0 0 / 50%)` | Heavy glass (cards, modals) |
| `--glass-border` | `oklch(0 0 0 / 8%)` | `oklch(1 0 0 / 12%)` | Glass element borders |
| `--glass-shadow` | `0 8px 32px oklch(0 0 0 / 8%)` | `0 8px 32px oklch(0 0 0 / 40%)` | Standard glass shadow |
| `--glass-shadow-heavy` | `0 25px 50px -12px oklch(0 0 0 / 15%)` | `0 25px 50px -12px oklch(0 0 0 / 25%)` | Heavy shadow for elevated elements |

### Glass Utility Classes

| Class | Properties | Use Case |
|-------|------------|----------|
| `.glass` | `background: var(--glass-bg)`, `backdrop-filter: blur(16px)`, `border: 1px solid var(--glass-border)` | Standard glass panels |
| `.glass-card` | `background: var(--glass-bg-heavy)`, `backdrop-filter: blur(12px)`, `border: 1px solid var(--glass-border)`, `box-shadow: var(--glass-shadow-heavy)` | Elevated glass cards |

### Glass Usage Guidelines

1. **Always ensure something to blur** - Glass effects need visible content behind them (gradient, image, or colored background)
2. **Prefer glass-card for interactive elements** - The heavier background provides better text contrast
3. **Combine with shadows** - Use `shadow-xl` or similar for depth
4. **Test both themes** - Glass appearance differs significantly between light and dark

## Accessibility

### Reduced Transparency

For users with `prefers-reduced-transparency`:
- Glass backgrounds become nearly opaque (95-98%)
- `backdrop-filter: blur()` is disabled

### No Backdrop Filter Support

For browsers without backdrop-filter support:
- Glass backgrounds fall back to solid `var(--card)` color
